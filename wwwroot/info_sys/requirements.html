What is a System boundary ?
  • Information systems- An Information System (IS) stores and processes data about some human activity to help manage or organise that activity. The IS is only part of a Human System.
  • Everything inside the system boundary constitutes the system.
  • The data flows that cross the system boundary constitute interaction with the environment.
  • The things outside the system that interact with it (actors) reside in the perceived environment.
What is a Statement of purpose and scope ?
  • A short concise statement of the primary purpose, function or role of the system being developed.
  • A touchstone for What is IN the System.
    • d
  • Write a statement of purpose for the Widget Warehouse Sales Order system.
  • The function of the Widget Warehouse Sales Order system is to manage customer orders from placement through to final payment. It does not include stock management functions.
What are Use cases ?
  • USE CASE - A use case is a formal way that represents how a business system interacts with its environment
  • Describes the system’s behaviour as it responds to a request from a stakeholder (the primary actor)
  • States a requirement for the system behaviour
  • The primary actor initiates a use case to achieve a goal
  • It describes the sequence of interactions to achieve the goal
  • Examples of a use case:
    • Primary actor: Customer
    • Goal: To get some goods they need
    • Scope: Widget Warehouse business
    • Main sequence:
      1. Customer rings up with order
      2. Warehouse dispatches the goods
      3. Customer signs the delivery note
      4. Warehouse sends an invoice
      5. Customer sends back a cheque
