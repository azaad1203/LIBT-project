Preparing Data for Analysis

Laying foundations for analysis
  • Before you start collecting data you need to have a plan you need to be sure of the following:
    • Why you are collecting the data
    • What data you need to collect
    • How you are going to use this data to answer those questions
    • Remember Research question and research methodology!
  • Why are you collecting data?
    • Satisfying some information need
    • Otherwise why both collecting it?
    • To answer a question:
      • “Which algorithm performs better? X or Y?”
      • “Which life-style factors predict happiness?”
      • “Do men sleep longer than women?”
How are you going to analyse the data
  • Which statistical test / chart is going to give you a clear answer to your question(s)?
  • What are the data requirements?
    • How much? (Scale), e.g test scores, time efficacy
    • What order? (Ordinal)
    • How many? (Nominal)
  • If you want to know if men sleep longer than women, you need a precise measure of sleep time
  • Do you sleep: a) Less than 5 hours b) 5 to 8 hours or c) 8 or more hours may not be a sufficiently sensitive measurement
  • On the other hand, if you need to know whether short-sleepers are happier than long sleepers then this graded measure may be sufficient
Preparing a codebook
  • A codebook is a summary description of how the data collected from your study will be/is represented in an SPSS data file
  • Ensures that you:
    • Enter data in a consistent and logical way.
    • Can understand the data table in the future.
  • It includes:
    • Definitions and labels for each variable
    • Numerical coding schemes for categorical and qualitative data
Coding qualitative data
  • Quantitative or qualitative?
    • Quantitative = numbers
    • Qualitative = ranks, classes, verbal responses
  • SPSS works with numbers
  • Different levels of measurement in SPSS
    • Nominal
    • Ordinal
    • Scale
  • Scale variables have values that represent ordered categories with a meaningful metric, so that distance comparisons between values are appropriate. E.g. Age, weight, hours, miles
  • Often data is not quantitative but still possible to encode numerically
    • Some variables are ordinal (values represent categories with some intrinsic ranking)
      • A Likert scale is ordered e.g. not at all, somewhat, a lot so we can assign the following codes: 1 = not at all, 2 = somewhat and 3 = a lot
    • Some variables are Nominal(values represent categories with no intrinsic ranking)
      • E.g. Marital status: 1 = single, 2 = steady, 3 = married, 4 = remarried
    • Values don’t tells us whether a case has more or less value than another case
    • Some data may not even present: obvious categories
      • Open-ended question responses are a good example
    • Need to scan through responses and extract common themes
Coding quantitative data
  • We will talk more about how to analyse data that have been collected quantitatively at the end of this semester. One of them is t-test.
  • We have shown for the past weeks about how to define your variables.
  • Small errors in data entry can cause significant distortions in your results. E.g. entering 91 when you meant 19
  • Subject responses might be invalid
    • Misunderstood the questions
    • Were not interested so answered randomly
  • Categorical variables
    • Nominal
      • Variables are categorised but with no order. Example: Gender (Male or female); Types of milk (Skimmed, semi-skimmed, or whole)
    • Ordinal
      • Variables that do have a natural order. Example: Ask patients to express the amount of pain they are feeling on a scale of 0 to 3, where 0 is none, 1 is low, 2 is medium, and 3 is high
    • Interval
      • Created from interval on a continuous scale. Example: Height (in kg) can be categorised as follows:- 0-20 , 21-40 , 41+
    • A continuous variable is one for which any value is possible:
      • For example, the variable ‘Time to debug errors in a program’ is continuous since it could take 5 minutes, 30 minutes, 500 minutes and so on, to debug errors.
      • Another example can be of temperature.
      • A Multiple choice test (50 marks with 1 mark for each question) is not continuous because it is not possible for someone to score 20.7, 90.5
      • A variable that is not continuous is called discrete
