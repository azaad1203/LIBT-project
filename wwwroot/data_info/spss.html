Key Concept

Statistical Package for the Social Sciences (SPSS)

Why do we use SPSS?
  • Data analysis is only one part of the research process
  • Once you have collected your data (quantitative and qualitative), they need to be analysed so as to answer the research questions that you addressed in your study
  • For this module, we will show you how to analyse the quantitative data.
    • The analysis of the quantitative data can be performed using a computer program called Statistical Package for the Social Sciences (SPSS)
  • Qualitative data can also be transformed into numbers before they are analysed in SPSS, but this is very rare in research.
Analysis of Quantitative data
  • How do we analyse the quantitative data so as to support us to answer the following types of research questions
    • RQ1: Overall, what are the students’ attitudes towards the parking facilities at Brunel University?
    • RQ2: Is there any correlation between mother tongue and Economics essay writing scores in relation to students studying in UK Universities?
    • RQ3: What are the effects of levels of Python experience on Python practical task scores when using a printed book (about Python) and when using audio/video instructional aids to perform python in the labs?
  • Example of RQ1:
    • Overall, I am happy with the parking space at Brunel
      • 1=strongly agree; 2=agree; 3=neutral; 4=disagree; 5=strongly agree
        20% strongly agree; 10% agree; 10 % neutral; 20% disagree and 40% strongly disagree
SPSS: Data Editor
  • Data are entered, edited and defined.
  • SPSS made up of two screens:
    1. The Data Viewer: the spreadsheet into which the data you gathered is entered. When you start up SPSS, the Data Viewer will be facing you
    2. The Variable Viewer: another spreadsheet but this one displays information about each of the variables and allows you to change that information.
      • It is the platform from which you provide for each variable such information as: the variable name; a variable label; and value labels
  • A spreadsheet grid in which data is entered
  • The columns represent variables – i.e., information about characteristics of each person in the gym study sample
  • Until data are entered:
    • Each column simply has var as its heading.
    • The rows represent cases, which can be people (as in the gym study example) or any unit of analysis
    • Each block in the grid is referred to as a ‘cell’.
    • Note also that when the data are in the SPSS spreadsheet, they will look different, e.g, 1 will be 1.00
  • Name
    • The name that you give to a variable, e.g. gender
    • Until you give a variable a name, it will be referred to as var00001, etc
    • When the variable has been given a name, it will appear in the column for that variable in the Data View window since it is generated from the Variable Viewer
  • Type
    • The default value for Type that will appear as you enter your first variable name is Numeric
    • To change it to other cases, for example ‘string’, simple click on the right hand side of the cell where there are three dots. This will display the option available.
    • If your variable include decimal places, then you need to make the necessary adjustments. We will come back later on decimal…
  • Width
    • The default value for Width is 8. This is usually sufficient for most data. If your variable has very large values you may need to change the default value.
  • Decimals
    • If your variable have decimal places, change this to suit your needs. If all your variables require decimal places, change this under Options (using the Data Tab).
  • Label
    • This is a long description that you can give to a variable but which is not restricted to eight characters.
    • Spaces can be used, e.g. reasons for visiting.
    • The Label will appear in any output you generate since it is generated from the Variable Viewer
  • Values
    • In the values column you can define the meaning of the values you have used to code your variables
    • E.g. for variable ‘gender’, we would attach the label male to 1 and female to 2
    • When you generate output, such as a frequency table or chart, the labels for each value will be presented
    • This makes the interpretation of output easier
    • It is generated from the Variable Viewer
  • Missing Values
    • When you do not have data for a particular variable when entering data for a case, you must specify how you are denoting missing values for that variable.
    • Missing values are generated from the Variable Viewer
  • Columns
    • The default column is usually set at 8. To make your data file smaller (so as to fit on the screen), you may choose to reduce the column width.
    • Allow enough space for the width of the variable name.
  • Align
    • The alignment of the columns is usually set at ‘right’ alignment. There is no need to change this.
  • Measure
    • The column heading ‘Measure’ refers to the level of measurement of each of your variable. The default is Scale, which refers to an interval or ratio level of measurement. If your variable consists of categories (e.g., sex), then click in the cell, and then on the arrow key that appears. Choose Nominal for categorical data, Ordinal if your data involves ranking.
